# 智能仓库管理系统 - 本地部署指南

## 📋 项目简介

这是一个基于 SpringBoot + Vue3 开发的智能仓库管理系统，包含用户管理、供应商管理、物料管理、仓位管理、出入库管理等核心功能。

**技术栈：**
- 后端：SpringBoot 2.7.18 + MyBatis-Plus + MySQL
- 前端：Vue 3 + Vite + Naive UI + ECharts

---

## 🛠️ 环境准备

在开始部署之前，请确保你的电脑已安装以下软件：

### 1. JDK 1.8
- **下载地址：** https://www.oracle.com/java/technologies/javase/javase8-archive-downloads.html
- **验证安装：** 打开命令行，输入 `java -version`，看到版本信息即表示安装成功

### 2. MySQL 数据库
- **推荐版本：** MySQL 5.7 或 8.0
- **下载地址：** https://dev.mysql.com/downloads/mysql/
- **或使用：** Navicat Premium、phpMyAdmin 等数据库管理工具

### 3. Node.js
- **推荐版本：** Node.js 16 或以上
- **下载地址：** https://nodejs.org/
- **验证安装：** 打开命令行，输入 `node -v` 和 `npm -v`，看到版本信息即表示安装成功

### 4. Maven（可选）
- **推荐版本：** Maven 3.6+
- **下载地址：** https://maven.apache.org/download.cgi
- **说明：** 如果使用 IDEA 开发工具，可以使用内置的 Maven

### 5. 开发工具（推荐）
- **后端：** IntelliJ IDEA（推荐）或 Eclipse
- **前端：** VS Code（推荐）或 WebStorm
- **数据库：** Navicat Premium 或 MySQL Workbench

## 📦 部署步骤

### 第一步：初始化数据库

1. **启动 MySQL 数据库服务**
   - Windows：在服务中启动 MySQL 服务
   - Mac/Linux：使用命令 `sudo service mysql start`

2. **创建数据库并导入数据**

   **方法一：使用 Navicat 等图形化工具**
   - 打开 Navicat，连接到本地 MySQL（账号：root，密码：123456）
   - 右键点击连接 → 运行 SQL 文件
   - 选择项目根目录下的 `init_warehouse.sql` 文件
   - 点击"开始"执行

   **方法二：使用命令行**
   ```bash
   # 进入项目根目录
   cd e:\bishe\基于SpringBoot与Vue的智能仓库管理系统设计与实现

   # 登录 MySQL（输入密码：123456）
   mysql -u root -p

   # 执行 SQL 脚本
   source init_warehouse.sql;

   # 退出 MySQL
   exit;
   ```

3. **验证数据库是否创建成功**
   - 数据库名称：`warehouse_db`
   - 应包含 6 张表：sys_user、wms_supplier、wms_material、wms_location、wms_inbound、wms_outbound
   - 已自动插入演示数据

---

### 第二步：启动后端服务

1. **打开后端项目**
   - 使用 IDEA 打开 `warehouse-backend` 文件夹
   - 等待 Maven 自动下载依赖（首次打开需要一些时间）

2. **配置数据库连接（如需修改）**
   - 打开文件：`warehouse-backend/src/main/resources/application.yml`
   - 确认数据库配置：
     ```yaml
     spring:
       datasource:
         url: jdbc:mysql://localhost:3306/warehouse_db?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai
         username: root
         password: 123456
     ```
   - 如果你的 MySQL 账号密码不同，请修改 username 和 password

3. **启动后端服务**

   **方法一：使用 IDEA 启动（推荐）**
   - 找到主启动类：`com.warehouse.WarehouseBackendApplication`
   - 右键点击 → Run 'WarehouseBackendApplication'
   - 看到控制台输出 "Started WarehouseBackendApplication" 表示启动成功

   **方法二：使用命令行启动**
   ```bash
   # 进入后端项目目录
   cd warehouse-backend

   # 使用 Maven 启动
   mvn spring-boot:run
   ```

4. **验证后端是否启动成功**
   - 浏览器访问：http://localhost:8080
   - 如果看到页面或返回数据，说明后端启动成功
   - 后端服务运行在 **8080 端口**

---

### 第三步：启动前端服务

1. **打开前端项目**
   - 使用 VS Code 打开 `warehouse-frontend` 文件夹

2. **安装依赖**
   ```bash
   # 进入前端项目目录
   cd warehouse-frontend

   # 安装依赖（首次运行需要执行）
   npm install
   ```

   **注意：** 如果 `npm install` 速度很慢，可以使用国内镜像：
   ```bash
   # 设置淘宝镜像
   npm config set registry https://registry.npmmirror.com

   # 再次安装
   npm install
   ```

3. **启动前端服务**
   ```bash
   # 启动开发服务器
   npm run dev
   ```

4. **验证前端是否启动成功**
   - 看到控制台输出：`Local: http://localhost:3000/`
   - 前端服务运行在 **3000 端口**

## ❓ 常见问题

### 1. 后端启动失败

**问题：** 提示数据库连接失败
```
com.mysql.cj.jdbc.exceptions.CommunicationsException: Communications link failure
```

**解决方案：**
- 检查 MySQL 服务是否启动
- 检查数据库账号密码是否正确
- 检查数据库 `warehouse_db` 是否已创建
- 检查 MySQL 端口是否为 3306

---

### 2. 前端启动失败

**问题：** 提示端口被占用
```
Port 3000 is in use
```

**解决方案：**
- 关闭占用 3000 端口的程序
- 或修改 `vite.config.js` 中的端口号：
  ```javascript
  server: {
    port: 3001  // 改为其他端口
  }
  ```

---

### 3. 前端无法访问后端接口

**问题：** 浏览器控制台提示 404 或网络错误

**解决方案：**
- 确认后端服务已启动（访问 http://localhost:8080 验证）
- 检查前端代理配置（`vite.config.js` 中的 proxy 配置）
- 检查浏览器控制台的具体错误信息

---

### 4. Maven 依赖下载失败

**问题：** IDEA 提示依赖下载失败

**解决方案：**
- 配置 Maven 国内镜像（阿里云镜像）
- 在 Maven 的 `settings.xml` 中添加：
  ```xml
  <mirror>
    <id>aliyun</id>
    <mirrorOf>central</mirrorOf>
    <url>https://maven.aliyun.com/repository/public</url>
  </mirror>
  ```

---

### 5. 登录后页面空白

**问题：** 登录成功但页面显示空白

**解决方案：**
- 打开浏览器开发者工具（F12）查看控制台错误
- 检查后端接口是否正常返回数据
- 清除浏览器缓存后重试

## 📝 注意事项

1. **启动顺序：** 必须先启动后端服务，再启动前端服务
2. **数据库：** 确保 MySQL 服务一直运行，否则后端会报错
3. **端口占用：** 如果 3000 或 8080 端口被占用，需要修改配置或关闭占用端口的程序
4. **防火墙：** 如果无法访问，检查防火墙是否拦截了端口
5. **浏览器：** 推荐使用 Chrome、Edge 等现代浏览器
