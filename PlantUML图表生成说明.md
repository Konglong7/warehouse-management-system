## 如何生成PNG图片

### 方法一：使用在线PlantUML编辑器（推荐）

1. 访问 http://www.plantuml.com/plantuml/uml/
2. 将 `.puml` 文件内容复制粘贴到编辑器中
3. 点击"Submit"按钮生成图片
4. 右键保存图片，或使用页面上的下载按钮

### 方法二：使用VS Code插件

1. 在VS Code中安装 "PlantUML" 插件
2. 打开 `.puml` 文件
3. 按 `Alt+D` 预览图表
4. 右键点击预览窗口，选择"Export Current Diagram"
5. 选择PNG格式导出

### 方法三：使用命令行工具

1. 下载 PlantUML jar 文件：
   ```bash
   curl -L -o plantuml.jar https://github.com/plantuml/plantuml/releases/download/v1.2024.3/plantuml-1.2024.3.jar
   ```

2. 生成图片：
   ```bash
   java -jar plantuml.jar "图3-3_系统用户角色权限用例图.puml"
   java -jar plantuml.jar "图4-2_系统功能模块图.puml"
   ```

3. 生成的PNG文件将保存在同一目录下

## 在Word中使用

1. 生成PNG图片后，在Word中插入图片
2. 设置图片格式：
   - 布局：嵌入型或四周型
   - 大小：根据页面宽度调整（建议宽度14-16cm）
   - 对齐：居中对齐
3. 添加图注：
   - 在图片下方插入文本框或直接输入
   - 格式：图3-3 系统用户角色权限用例图
   - 字体：宋体，五号，加粗，居中

## 图注说明示例

### 图3-3 系统用户角色权限用例图

如图3-3所示，系统设计了普通用户和管理员两种角色。管理员继承普通用户的所有权限，并额外拥有基础数据管理权限。普通用户可以进行日常的入库、出库、查询等业务操作，但不能修改物料、供应商、仓位等基础数据。管理员除了拥有普通用户的所有权限外，还可以对物料、供应商、仓位进行增删改操作，并能够管理用户账户和查看所有用户的操作记录。这种权限设计既保证了业务操作的灵活性，又避免了误操作导致基础数据被破坏。

### 图4-2 系统功能模块图

如图4-2所示，系统采用模块化设计，分为用户管理模块、基础数据管理模块、业务操作模块、查询统计模块和系统支撑模块五大部分。用户管理模块负责用户认证和权限控制；基础数据管理模块包括物料管理、供应商管理和仓位管理三个子模块；业务操作模块包括入库管理和出库管理，实现核心业务流程；查询统计模块提供库存查询和数据统计功能；系统支撑模块提供数据库管理、日志管理和操作记录等基础服务。各模块之间通过清晰的接口进行交互，形成完整的业务闭环。
