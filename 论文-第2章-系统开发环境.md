# 第2章 系统开发环境

## 2.1 开发语言

后端开发选择了Java语言。Java是一种面向对象的编程语言，具有跨平台的特性。在企业级应用开发中，Java一直占据主导地位。对于本系统来说，仓库管理涉及到库存数据的准确性，选择Java这种成熟稳定的语言是比较合适的。本系统使用的是Java 8版本。

前端开发使用JavaScript语言。JavaScript是Web开发的标准语言，所有的浏览器都支持。本系统使用的是ES6及以上的语法标准，箭头函数、解构赋值、模块化等新特性让代码写起来更简洁。

## 2.2 开发框架

### 2.2.1 后端框架选型

后端框架的选择上，我对比了几种常见的Java Web框架。

表2-1 后端框架对比

| 框架 | 学习成本 | 开发效率 | 配置复杂度 |
|------|---------|---------|-----------|
| SpringBoot | 中等 | 高 | 低 |
| SSM | 较高 | 中等 | 高 |
| Spring Cloud | 高 | 高 | 很高 |

最终选择SpringBoot主要是因为它简化了Spring应用的配置，采用"约定优于配置"的思想。SpringBoot内置了Tomcat服务器，不需要单独部署。在本系统中，SpringBoot主要用于构建RESTful API接口。

### 2.2.2 前端框架选型

前端框架方面，我对比了目前主流的几个框架。

表2-2 前端框架对比

| 框架 | 学习曲线 | 开发效率 | 性能 |
|------|---------|---------|------|
| Vue 3 | 平缓 | 高 | 优秀 |
| React | 较陡 | 高 | 优秀 |
| Angular | 陡峭 | 中等 | 良好 |

Vue 3是本系统选择的前端框架。Vue的学习曲线比较平缓，模板语法更接近HTML，上手比较快。Vue 3相比Vue 2性能更好，而且引入了Composition API，让代码的组织更灵活。

持久层框架选择了MyBatis-Plus。这个框架是在MyBatis基础上做的增强，提供了BaseMapper接口，继承这个接口就自动拥有了增删改查的方法，不需要写SQL。

## 2.3 数据库

数据库选择了MySQL。MySQL是目前使用最广泛的开源关系型数据库。选择MySQL主要考虑了这几点：MySQL是免费开源的；MySQL的性能不错，对于中小型应用来说完全够用；MySQL的生态很成熟，有Navicat等好用的管理工具。本系统使用的是MySQL 8.0版本。

## 2.4 开发工具

后端开发使用IntelliJ IDEA，这是JetBrains公司开发的Java IDE，功能非常强大。前端开发使用Visual Studio Code，这是微软开发的一款轻量级代码编辑器。数据库管理使用Navicat Premium，可以可视化地设计表结构。接口测试使用Postman，专门用于测试API接口。

## 2.5 其他技术

前端构建工具使用Vite，这是Vue团队开发的新一代前端构建工具，启动速度很快。后端构建工具使用Maven，可以管理项目依赖、编译代码、打包部署等。

前端UI组件库使用Naive UI，这是一个Vue 3的组件库，提供了表格、表单、对话框等常用组件。数据可视化使用ECharts，这是百度开源的一个图表库。在本系统中，主要用到了柱状图和饼图。

## 2.6 运行环境

系统的运行环境要求如下：

表2-3 运行环境要求

| 环境类型 | 名称 | 版本要求 |
|---------|------|---------|
| 操作系统 | Windows/Linux/Mac | 不限 |
| JDK | Java Development Kit | 1.8及以上 |
| 数据库 | MySQL | 5.7或8.0 |
| 浏览器 | Chrome/Edge/Firefox | 最新版本 |

整体来说，系统的运行环境要求不高，普通的个人电脑就可以运行。
