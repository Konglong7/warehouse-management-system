# 第3章 系统分析

系统分析是软件开发的重要阶段，需要明确系统要做什么，为谁做，怎么做。这一章主要从可行性分析、功能需求分析、用户角色权限和业务流程几个方面对智能仓库管理系统进行详细分析。

## 3.1 系统可行性分析

在正式开发系统之前，需要对项目的可行性进行评估，确保项目能够顺利完成。可行性分析主要从技术可行性、经济可行性和操作可行性三个方面进行。

### 3.1.1 技术可行性

技术可行性主要考虑系统开发所需的技术是否成熟，开发团队是否具备相应的技术能力。

本系统采用的技术栈都是目前比较成熟的主流技术。后端使用SpringBoot框架，这个框架在企业开发中应用很广泛，网上有大量的学习资料和开源项目可以参考。前端使用Vue 3框架，Vue的学习曲线比较平缓，而且官方文档写得很详细，遇到问题基本上都能找到解决方案。数据库使用MySQL，这是最常用的关系型数据库，稳定性和性能都有保障。

在开发过程中，虽然会遇到一些技术难点，比如前后端数据交互、库存数据一致性保证等，但这些问题都有成熟的解决方案。通过查阅文档、搜索资料、请教老师，这些技术问题都是可以解决的。

从技术角度来看，本系统的开发是可行的。

### 3.1.2 经济可行性

经济可行性主要考虑系统开发和运行的成本是否在可接受范围内。

本系统使用的都是开源免费的技术和工具。SpringBoot、Vue、MySQL都是开源软件，不需要购买许可证。开发工具方面，IntelliJ IDEA有社区版可以免费使用，VS Code也是免费的。虽然Navicat是商业软件，但可以使用试用版或者用免费的MySQL Workbench代替。

系统的运行环境要求也不高，普通的个人电脑就可以运行。如果要部署到服务器上，可以选择阿里云、腾讯云这些云服务商的学生优惠套餐，成本也很低。

对于中小企业来说，开发和维护这样一套系统的成本远低于购买商业软件。而且系统的源代码完全掌握在自己手里，可以根据实际需求进行定制开发。

从经济角度来看，本系统的开发是可行的。

### 3.1.3 操作可行性

操作可行性主要考虑系统是否易于使用，用户是否能够快速上手。

本系统的设计思路是尽量简化操作流程，降低使用门槛。界面设计采用了现代化的UI组件库，布局清晰，操作直观。物料编码、入库单号这些都是系统自动生成的，用户不需要手工输入。表单填写时会有必填项提示，输入错误时会有明确的错误提示。

系统的主要用户是仓库管理人员，他们可能没有太多的计算机基础。考虑到这一点，系统的功能设计都是围绕实际业务流程展开的，比如入库就是选择物料、选择仓位、输入数量，这和实际的入库操作流程是一致的。用户不需要学习复杂的操作，只要熟悉自己的业务就能使用系统。

从操作角度来看，本系统的开发是可行的。

## 3.2 功能需求分析

功能需求分析是系统分析的核心内容，需要明确系统要实现哪些功能。根据对仓库管理业务的调研和分析，本系统需要实现以下功能模块。

### 3.2.1 用户管理功能

用户管理是系统的基础功能，负责用户的登录认证和权限控制。

用户登录功能是系统的入口。用户需要输入用户名和密码才能进入系统。系统会验证用户身份，验证通过后才允许访问系统的其他功能。登录成功后，系统会记录用户的登录状态，用户在一定时间内不需要重复登录。

用户信息管理功能允许用户查看和修改自己的个人信息，包括昵称、手机号等。管理员还可以管理所有用户的信息，包括新增用户、修改用户信息、删除用户等。

密码修改功能允许用户修改自己的登录密码。为了保证安全，修改密码时需要先输入原密码进行验证。

### 3.2.2 物料管理功能

物料管理是系统的核心功能之一，负责管理仓库中所有物料的基本信息。

物料信息管理功能包括物料的新增、修改、删除和查询。每个物料都有唯一的物料编码，还包括物料名称、分类、单位、供应商、预警库存、当前库存等信息。物料编码在系统中是唯一的，不能重复。

物料查询功能支持按物料名称或编码进行模糊搜索，方便用户快速找到需要的物料。查询结果以表格形式展示，可以看到物料的所有信息。

库存预警功能可以帮助管理员及时了解库存不足的物料。每个物料都可以设置一个预警库存值，当物料的当前库存低于这个值时，系统会在首页显示预警提示。

### 3.2.3 供应商管理功能

供应商管理功能用于维护供应商的基本信息。供应商信息包括供应商编码、供应商名称、联系人、联系电话、地址等。供应商编码同样需要保证唯一性。

供应商信息可以与物料信息关联，记录每个物料的供应商是谁。这样在需要补货时，可以快速找到对应的供应商联系方式。

### 3.2.4 仓位管理功能

仓位管理功能用于维护仓库的仓位信息。仓位信息包括仓位编码、仓位名称、所属仓库、容量、状态等。仓位状态分为可用、占用、禁用三种。

在入库和出库操作时，需要指定物料存放的仓位。通过仓位管理，可以清楚地知道每个仓位的使用情况，合理安排仓库空间。

### 3.2.5 入库管理功能

入库管理是系统的核心业务功能，处理物料的入库操作。入库操作需要记录入库的物料、数量、仓位、操作人、入库时间等信息。系统会自动生成入库单号，方便后续查询和追溯。

入库操作完成后，系统会自动更新物料的当前库存。比如某个物料原来库存是100，入库50后，库存自动变成150。这个更新过程是自动完成的，不需要人工干预。

入库记录可以查询和统计，支持按时间范围、物料名称、操作人等条件进行筛选。

### 3.2.6 出库管理功能

出库管理与入库管理相对应，处理物料的出库操作。出库操作的流程和入库类似，但需要增加库存校验功能。

在出库之前，系统会先检查物料的当前库存是否充足。如果出库数量大于当前库存，系统会提示"库存不足"，不允许出库。这样可以避免库存出现负数的情况。

出库操作完成后，系统会自动减少物料的库存。出库记录同样可以查询和统计。

### 3.2.7 库存查询功能

库存查询功能用于查看所有物料的库存状态。查询结果显示物料的编码、名称、分类、当前库存、预警库存、库存状态等信息。

库存状态根据当前库存和预警库存的关系自动计算。当前库存低于预警库存时显示"预警"，提醒管理员需要补货。库存充足时显示"充足"，库存适中时显示"正常"。

通过不同颜色的标签，可以直观地看出哪些物料需要关注。

### 3.2.8 数据统计功能

数据统计功能用于展示系统的各种统计数据。首页会显示总库存、今日入库、今日出库、预警物料数量等关键指标。

系统还提供了图表展示功能，使用柱状图展示库存统计，使用饼图展示物料分类占比。这些图表可以帮助管理员直观地了解仓库的运营情况。

## 3.3 用户角色权限分析

系统设计了两种用户角色：普通用户和管理员。不同角色拥有不同的权限。

### 3.3.1 普通用户权限

普通用户是系统的主要使用者，通常是仓库的操作人员。普通用户的权限包括：

- 登录系统，查看系统首页的统计数据和图表
- 查看物料列表，搜索物料信息
- 进行入库操作，记录物料入库
- 进行出库操作，记录物料出库
- 查看入库记录和出库记录
- 查看库存信息，了解物料库存状态
- 修改个人信息，修改登录密码

普通用户不能进行物料信息的新增、修改、删除操作，也不能管理供应商和仓位信息。这样可以避免误操作导致基础数据被破坏。

### 3.3.2 管理员权限

管理员是系统的管理者，通常是仓库主管或系统管理员。管理员拥有系统的所有权限，包括：

- 普通用户的所有权限
- 新增、修改、删除物料信息
- 新增、修改、删除供应商信息
- 新增、修改、删除仓位信息
- 管理用户信息，包括新增用户、修改用户信息、删除用户
- 查看所有用户的操作记录

管理员可以对系统的基础数据进行维护，确保系统数据的准确性和完整性。

## 3.4 业务流程分析

业务流程分析主要描述系统的主要业务流程，明确各个环节的操作步骤。

### 3.4.1 用户登录流程

用户登录是使用系统的第一步。用户打开系统后，首先看到的是登录页面。用户输入用户名和密码，点击登录按钮。系统验证用户名和密码是否正确，如果正确则登录成功，跳转到系统首页；如果错误则提示"用户名或密码错误"，用户需要重新输入。

登录成功后，系统会保存用户的登录状态。用户在使用系统的过程中不需要重复登录。如果用户长时间没有操作，登录状态会过期，需要重新登录。

### 3.4.2 物料入库流程

物料入库是仓库管理的核心业务之一。入库流程如下：

用户点击"入库管理"菜单，进入入库管理页面。点击"新增入库"按钮，弹出入库信息录入对话框。用户选择要入库的物料，选择存放的仓位，输入入库数量，填写备注信息。点击保存按钮，系统验证输入的数据是否合法。如果数据合法，系统保存入库记录，自动生成入库单号，同时更新物料的库存数量。入库成功后，系统提示"入库成功"，关闭对话框，刷新入库记录列表。

整个入库流程操作简单，用户只需要几步就能完成入库操作。

### 3.4.3 物料出库流程

物料出库的流程和入库类似，但增加了库存校验环节。出库流程如下：

用户点击"出库管理"菜单，进入出库管理页面。点击"新增出库"按钮，弹出出库信息录入对话框。用户选择要出库的物料，选择仓位，输入出库数量，填写备注信息。点击保存按钮，系统首先检查物料的当前库存是否充足。如果出库数量大于当前库存，系统提示"库存不足"，不允许出库。如果库存充足，系统保存出库记录，自动生成出库单号，同时减少物料的库存数量。出库成功后，系统提示"出库成功"，关闭对话框，刷新出库记录列表。

库存校验是出库流程的关键环节，可以避免超出库存的出库操作。

### 3.4.4 库存查询流程

库存查询是管理员经常使用的功能。查询流程如下：

用户点击"库存查询"菜单，进入库存查询页面。系统自动加载所有物料的库存信息，以表格形式展示。表格中显示物料编码、物料名称、分类、当前库存、预警库存、库存状态等信息。用户可以在搜索框中输入物料名称或编码，系统会实时过滤显示匹配的物料。用户还可以点击"预警物料"按钮，快速筛选出所有库存不足的物料。

通过库存查询，管理员可以随时了解仓库的库存情况，及时发现需要补货的物料。

通过以上分析，明确了系统的功能需求、用户角色权限和业务流程。这些分析为后续的系统设计和实现提供了依据。
